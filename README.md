# Edit Studio

브라우저 안에서 영상 파일을 불러와 자막, 자유 텍스트, 클릭/터치 이펙트 오버레이를 편집하고, SRT/WebVTT/ASS 또는 효과가 입혀진 MP4로 내보내는 로컬 웹 앱입니다.

영상 파일은 서버로 업로드하지 않고 브라우저 메모리와 FFmpeg WASM 안에서 처리됩니다.

## 실행

```bash
npm install
npm run dev
```

개발 서버가 뜨면 표시된 로컬 주소를 브라우저에서 엽니다.

## 빠른 시작

1. 상단 `가져오기`에서 영상 파일을 선택하거나 화면에 드래그 앤 드롭합니다.
2. 우측 패널에서 `자막`, `텍스트`, `효과`를 추가합니다.
3. 미리보기에서 텍스트와 효과의 위치/크기를 직접 조절합니다.
4. 하단 타임라인에서 시작/종료 시간, 영상 조각, 자막/텍스트/효과 구간을 확인합니다.
5. 상단 `프로젝트 저장`으로 JSON 작업 파일을 저장하거나, `MP4 저장`으로 결과 영상을 내보냅니다.

## 주요 기능

- 비디오 파일 드래그 앤 드롭 또는 파일 선택.
- 미디어 보관함에서 여러 영상/이미지/오디오 소스 관리.
- 영상 조각 분할, 구간 제거, 클립별 속도 조절, Fade/Slide 전환 설정.
- 영상 조각별 위치, 확대/축소, 회전, 불투명도, crop 편집.
- 이미지 클립을 타임라인에 배치하고 미리보기에서 위치/크기 조절.
- Transform 키프레임 1차 지원.
- 자막 추가, 복제, 삭제, 시작/종료 시간 편집.
- 자유 텍스트 오버레이 추가, 위치/크기/스타일 편집.
- 텍스트 스타일 프리셋, 굵기, 기울임, 밑줄, 정렬, 3x3 고정 위치 프리셋.
- 텍스트/자막 배경 사용 토글, 배경 색상, 투명도 슬라이더.
- 텍스트/자막 외곽선 사용 토글, 외곽선 색상, 두께 조절.
- 추천 한국어 폰트 다운로드 링크와 TTF/OTF 폰트 가져오기
- 터치 링, 클릭 점, 펄스, 스포트라이트, 스와이프, 타겟 포커스, 마우스 포인터, 손가락 터치 이펙트 오버레이 추가와 위치/색상/크기 편집
- SRT/WebVTT 가져오기 및 SRT/WebVTT/ASS 내보내기
- 프로젝트 JSON 저장/불러오기
- 편집 Undo/Redo
- 브라우저 자동 저장/복구
- 첫 방문 튜토리얼과 상단 도움말 패널, 튜토리얼 다시보기
- 기본 단축키: Space 재생/정지, A 자막 추가, T 텍스트 추가, E 효과 추가, Delete 삭제, Cmd/Ctrl+Z 되돌리기, Cmd/Ctrl+Shift+Z 또는 Cmd/Ctrl+Y 다시 적용
- `ffmpeg.wasm` 기반 MP4 burn-in export

## 컷 편집 사용법

- `분할`: 현재 플레이헤드 위치에서 선택된 영상 조각을 둘로 나눕니다.
- `IN`, `OUT`: 제거하거나 확인할 구간의 시작/종료 지점을 지정합니다.
- `구간 제거`: IN/OUT 사이의 불필요한 구간을 제거하고 뒤 조각을 앞으로 자동으로 붙입니다.
- `속도`: 선택한 영상 조각의 재생 속도를 `0.25x`부터 `4x`까지 조절합니다.
- `전환`: 인접한 영상 조각 사이에 Fade 또는 Slide 전환을 붙입니다.

현재 버전은 미디어 보관함에 여러 영상 소스를 가져와 조각으로 배치할 수 있습니다. 다만 유료 편집툴 수준의 고급 멀티캠/프록시/전용 소스 모니터 워크플로우는 아직 포함되어 있지 않습니다.

## 타임라인 사용법

- 일반 휠/트랙패드: 타임라인 좌우 이동.
- `Shift + wheel`: 빠른 좌우 이동.
- `Ctrl/Cmd + wheel`: 커서 위치 기준 확대/축소.
- `Fit`: 전체 편집 길이를 한 화면에 맞춤.
- 줌 슬라이더와 `- / +`: 확대 배율 조절.
- 트랙 경계 드래그: 영상, 자막, 텍스트, 효과 트랙 높이 조절.
- 플레이헤드 드래그 또는 빈 시간축 클릭: 원하는 시간으로 이동.

## 텍스트 편집 사용법

- `텍스트 추가` 후 미리보기에서 드래그하면 위치가 바뀝니다.
- 선택된 텍스트의 모서리/변 핸들을 드래그하면 가로/세로 크기를 직접 조절할 수 있습니다.
- 우측 `고정 위치` 3x3 버튼으로 좌측 상단, 정중앙, 우측 하단 같은 위치에 빠르게 배치할 수 있습니다.
- `스타일 프리셋`으로 클린 타이틀, 캡션 박스, 네온 포인트, 에디터 라벨 스타일을 한 번에 적용할 수 있습니다.
- 배경과 외곽선은 각각 사용 여부를 켠 뒤 색상과 세부 값을 조절합니다.

## 프로젝트 저장과 초기화

- `프로젝트 저장`: 자막, 텍스트, 효과, 컷 편집 데이터, 전환 정보를 JSON으로 저장합니다.
- `프로젝트 불러오기`: 저장한 JSON 작업 파일을 다시 불러옵니다.
- 편집 내용은 브라우저에 자동 저장되고, 가능하면 원본 영상 파일도 IndexedDB에 보관해 다음 방문 시 자동 연결합니다.
- 브라우저 저장공간 제한이나 다른 컴퓨터에서 여는 경우에는 원본 영상이 자동 연결되지 않을 수 있습니다. 이때 화면의 `다시 연결` 버튼으로 같은 영상을 선택하면 이어서 작업할 수 있습니다.
- `초기화`: 현재 영상, 레이어, 편집 데이터, 렌더 결과를 비웁니다.

## MP4 내보내기

- `720p 빠른 렌더`: 확인용으로 빠르게 결과를 만들 때 적합합니다.
- `원본 해상도`: 최종 저장에 적합하지만 영상 길이와 해상도에 따라 오래 걸릴 수 있습니다.
- `맞춤 방식`: `꽉 채우기`, `전체 보이기`, `늘리기` 중 선택합니다. `전체 보이기`는 원본 비율을 유지하고 남는 공간을 검은색 레터박스로 채웁니다.
- Chrome/Edge에서는 MP4 저장 전에 파일 이름과 저장 위치를 직접 선택할 수 있습니다.
- 첫 렌더링 시 FFmpeg WASM 엔진을 브라우저에 준비하므로 대기 시간이 생길 수 있습니다.
- 렌더링 중 문제가 생기면 현재 코덱, 영상 길이, 브라우저 메모리 한계의 영향을 받을 수 있습니다.

## 참고

- 내부 구조와 라이브러리 역할은 [Edit Studio 구조 설명서](docs/project-architecture.md)를 참고하세요.
- MP4 내보내기는 브라우저에서 FFmpeg WASM을 실행하므로 긴 영상은 시간이 걸릴 수 있습니다.
- 편집 중인 프로젝트가 있으면 탭을 닫을 때 브라우저 기본 경고가 표시됩니다.
- 한국어 렌더링은 `public/fonts/AppleGothic.ttf`를 사용합니다.
- 가져온 TTF/OTF 폰트는 현재 브라우저 세션에서 미리보기와 MP4 내보내기에 함께 사용됩니다. 프로젝트 JSON에는 폰트 이름만 저장되므로, 다른 세션에서 같은 폰트를 쓰려면 파일을 다시 가져와야 합니다.
- 기본 export는 720p 빠른 렌더이며, 원본 해상도 옵션도 제공합니다.
- 브라우저 기반 렌더링이라 매우 큰 원본 영상은 메모리 한계에 걸릴 수 있습니다. 렌더가 길어질 때는 취소 후 720p 빠른 렌더를 먼저 확인하는 흐름을 권장합니다.
- ASS subtitle burn-in은 CSS 미리보기와 완전히 같은 렌더러가 아니므로, 그림자/배경 박스가 브라우저와 아주 미세하게 다를 수 있습니다.
