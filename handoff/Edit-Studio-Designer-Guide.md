# Edit Studio 디자이너 사용설명서

## 1. 이 앱은 무엇인가요?

Edit Studio는 로컬 브라우저에서 실행되는 영상 편집 웹 앱입니다.

디자이너는 영상 파일을 브라우저에 불러온 뒤 자막, 텍스트, 클릭/터치 효과를 얹고, 필요 없는 구간을 잘라낸 후 MP4로 저장할 수 있습니다. 영상은 별도 서버로 업로드되지 않고 현재 컴퓨터의 브라우저 안에서만 처리됩니다.

## 2. 실행 전 준비물

필요한 것:

- Chrome 또는 Edge 최신 버전
- Node.js 20 이상
- 전달받은 Edit Studio 압축 파일

Node.js가 설치되어 있는지 확인하려면 터미널에서 아래 명령을 입력합니다.

```bash
node -v
npm -v
```

버전이 표시되면 설치되어 있는 상태입니다. 명령을 찾을 수 없다고 나오면 Node.js를 먼저 설치해야 합니다.

## 3. 처음 실행하기

압축을 푼 폴더에서 터미널을 열고 아래 순서대로 실행합니다.

```bash
npm install
```

이 명령은 처음 한 번만 실행하면 됩니다. 앱 실행에 필요한 라이브러리를 내려받습니다.

그 다음 개발 서버를 실행합니다.

```bash
npm run dev
```

터미널에 로컬 주소가 표시되면 브라우저에서 아래 주소를 엽니다.

```text
http://127.0.0.1:5173/
```

서버를 끄려면 터미널에서 `Ctrl + C`를 누릅니다.

## 4. 미리 빌드된 화면 확인하기

전달 패키지에는 `dist` 폴더가 포함되어 있습니다. 소스 수정 없이 빌드된 결과만 확인하려면 아래 명령을 사용할 수 있습니다.

```bash
npm install
npm run preview
```

터미널에 표시되는 `http://127.0.0.1:4173/` 같은 주소를 브라우저에서 열면 됩니다.

## 5. 기본 작업 흐름

1. 상단 `가져오기`에서 영상 파일을 선택합니다.
2. 우측 패널에서 `자막`, `텍스트`, `효과` 탭을 선택합니다.
3. 필요한 항목을 추가합니다.
4. 미리보기 화면에서 텍스트와 효과의 위치와 크기를 조절합니다.
5. 하단 타임라인에서 시작/종료 시간과 영상 조각을 확인합니다.
6. 상단 `MP4 저장`으로 최종 영상을 저장합니다.

## 6. 영상 컷 편집

- `분할`: 현재 플레이헤드 위치에서 선택된 영상 조각을 둘로 나눕니다.
- `IN`: 제거할 구간의 시작점을 지정합니다.
- `OUT`: 제거할 구간의 끝점을 지정합니다.
- `구간 제거`: IN/OUT 사이를 삭제하고 뒤 조각을 앞으로 붙입니다.
- `속도`: 선택한 영상 조각의 속도를 조절합니다.
- `전환`: 인접한 영상 조각 사이에 Fade 또는 Slide 효과를 넣습니다.

현재 버전은 하나의 원본 영상 안에서 조각을 만들고 이어 붙이는 방식입니다. 여러 개의 영상 파일을 하나로 합치는 기능은 아직 포함되어 있지 않습니다.

## 7. 타임라인 사용법

- 마우스 휠 또는 트랙패드: 좌우 이동
- `Shift + wheel`: 빠른 좌우 이동
- `Ctrl/Cmd + wheel`: 커서 위치 기준 확대/축소
- `Fit`: 전체 길이를 한 화면에 맞춤
- 줌 슬라이더 또는 `- / +`: 확대 배율 조절
- 트랙 경계 드래그: 영상, 자막, 텍스트, 효과 영역 높이 조절
- 플레이헤드 드래그 또는 빈 시간축 클릭: 원하는 시간으로 이동

긴 영상은 먼저 `Fit`으로 전체 구조를 본 뒤 필요한 구간에서 확대하는 방식이 편합니다.

## 8. 자막과 텍스트

자막:

- 일반적인 하단 자막, 중앙 자막, 상단 자막을 만들 때 사용합니다.
- SRT/WebVTT 파일을 가져오거나 내보낼 수 있습니다.

텍스트:

- 제목, 라벨, 강조 문구, 화면 설명처럼 자유롭게 배치할 문구에 사용합니다.
- 미리보기에서 드래그해 위치를 바꿀 수 있습니다.
- 선택된 텍스트의 핸들을 드래그해 가로/세로 크기를 조절할 수 있습니다.
- `고정 위치` 3x3 버튼으로 좌측 상단, 정중앙, 우측 하단 같은 위치에 빠르게 배치할 수 있습니다.
- `스타일 프리셋`으로 클린 타이틀, 캡션 박스, 네온 포인트, 에디터 라벨 스타일을 적용할 수 있습니다.
- 배경과 외곽선은 각각 사용 여부를 켠 뒤 색상, 투명도, 두께를 조절합니다.

## 9. 클릭/터치 효과

효과 탭에서 클릭 포인터, 손가락 터치, 터치 링, 펄스, 스포트라이트 같은 오버레이를 추가할 수 있습니다.

- 효과는 미리보기에서 위치를 이동할 수 있습니다.
- 크기 핸들로 크기를 조절할 수 있습니다.
- 클릭 포인터와 손가락 터치 효과는 짧게 한 번 눌리고 사라지는 형태가 기본입니다.
- 효과 아이템을 다시 선택하면 애니메이션을 재생해 확인할 수 있습니다.

## 10. 프로젝트 저장과 불러오기

`프로젝트 저장`은 현재 작업 상태를 JSON 파일로 저장합니다.

저장되는 것:

- 자막
- 텍스트
- 효과
- 영상 조각 정보
- 속도와 전환 정보
- 스타일 정보

저장되지 않는 것:

- 원본 영상 파일 자체
- 사용자가 따로 가져온 폰트 파일 자체

같은 브라우저에서는 편집 내용이 자동 저장되고 원본 영상도 자동 복구를 시도합니다. 브라우저 저장공간 제한이 있거나 다른 컴퓨터에서 프로젝트 JSON을 열 때는 화면에 표시되는 `다시 연결` 버튼으로 같은 원본 영상을 선택하면 이어서 작업할 수 있습니다.

편집 중인 프로젝트가 있을 때 탭을 닫으면 브라우저 기본 경고가 표시됩니다. 경고 문구는 브라우저 정책상 앱에서 마음대로 바꿀 수 없습니다.

## 11. MP4 저장

상단에서 렌더 프리셋을 선택할 수 있습니다.

- `720p 빠른 렌더`: 빠른 확인용
- `원본 해상도`: 최종 저장용

첫 MP4 저장은 FFmpeg WASM 엔진을 준비하느라 시간이 걸릴 수 있습니다. 긴 영상이나 고해상도 영상은 브라우저 메모리 한계 때문에 오래 걸릴 수 있으므로, 먼저 720p 빠른 렌더로 확인하는 것을 권장합니다.

Chrome 또는 Edge에서는 MP4 저장 버튼을 누른 뒤 파일 이름과 저장 위치를 직접 선택할 수 있습니다. 지원하지 않는 브라우저에서는 파일 이름을 입력한 뒤 기본 다운로드 폴더로 저장됩니다.

## 12. 문제 해결

화면이 열리지 않을 때:

- `npm install`을 실행했는지 확인합니다.
- `npm run dev`가 꺼져 있지 않은지 확인합니다.
- 브라우저 주소가 `http://127.0.0.1:5173/`인지 확인합니다.

포트가 이미 사용 중일 때:

- 터미널에 다른 주소가 표시될 수 있습니다.
- 표시된 주소를 그대로 브라우저에 붙여 넣으면 됩니다.

MP4 저장이 오래 걸릴 때:

- 첫 실행은 FFmpeg 엔진 준비 시간이 필요합니다.
- 긴 영상은 720p 빠른 렌더로 먼저 확인합니다.
- 브라우저 탭을 닫거나 새로고침하면 렌더링이 중단됩니다.

프로젝트를 열었는데 영상이 없을 때:

- 화면 상단 또는 미리보기 영역의 `다시 연결` 버튼을 누릅니다.
- 안내에 표시된 파일명과 같은 원본 영상을 선택합니다.
- 파일명이 바뀐 경우에도 연결은 가능하지만, 다른 영상이면 컷 위치와 자막 타이밍이 맞지 않을 수 있습니다.

## 13. 디자이너 확인 체크리스트

- 앱이 로컬 주소에서 정상 실행되는가?
- 영상 파일을 가져올 수 있는가?
- 텍스트를 추가하고 고정 위치 버튼으로 배치할 수 있는가?
- 텍스트 배경/외곽선을 켜고 끌 수 있는가?
- 타임라인 확대/축소와 좌우 이동이 자연스러운가?
- 영상 조각을 분할하고 구간 제거할 수 있는가?
- MP4 저장 버튼을 눌렀을 때 렌더링 상태가 표시되는가?
- 저장된 MP4가 정상 재생되는가?
